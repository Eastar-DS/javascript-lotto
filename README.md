# 2단계 - 웹 기반 로또 게임 구현 기능 목록
1. [x] 공통 css 지정 
    - 폰트
    - 색상
2. [x] UI 구성
    1. [x] 헤더에 스타일 적용
        - 높이, 색
    2. [x] 푸터 copyright 작성, 스타일 적용
    3. [X] 구입 전
        - [x] 타이틀
        - [x] 구입할 금액 입력해주라는 텍스트
        - [x] 구입 금액 입력 칸
        - [] 에러 메세지 보여주 곳
        - [x] 구입 버튼
    4. [x] 구입 후 
        - [x] 구입 전 구성 요소는 그대로
        - [x] 몇 개 구입했는지 알려주는 텍스트
        - [x] 구입한 로또 번호 출력
        - [x] 지난 주 당첨 번호 6개와 보너스 번호 1개를 입력해주라는 텍스트
        - [x] 당첨 번호, 보너스 번호 각 입력 칸 위에 텍스트
        - [x] 당첨 번호 입력 칸 6개(1개의 숫자를 입력할 수  있는   칸 6개)
        - [x] 보너스 번호 입력 칸 1개
        - [] 입력 칸 밑에 에러 메세지 보여주는 텍스트
        - [x] 결과 확인하기 버튼
    5. [x] 당첨 통계 모달
        - [x] 당첨 모달을 닫을 수 있는 `X`버튼
        - [x] 당첨 통계 타이틀
        - [x] 일치 갯수, 당첨금, 당첨 갯수를 알려주는 표
        - [x] 수익률을 알려주는 텍스트
        - [x] 다시 시작하기 버튼

3. 동작 구성
      1. [x] 구입 전
          - [x] 구입 금액을 입력하고 구입 버튼을 클릭하면 로또를 구매한다.
            - [x] 구입 금액을 검증한다 (alert로 에러 메세지 띄우기)
          - [x] 구입 금액 입력 칸을 비운다
          - [x] 구입 후 UI를 보여준다.
          - [ ] 구입 후 구입 버튼 비활성화
      2. [x] 구입 후
          - [x] 몇 개 구입했는지 알려주는 텍스트 출력
          - [x] 구입한 로또 번호 출력
          - [x] 지난 주 당첨 번호 6개와 보너스 번호 1개를 입력해주라는 텍스트 출력
          - [x] 당첨 번호, 보너스 번호 각 입력 칸 위에 텍스트 출력
          - [x] 당첨 번호 6개와 보너스 번호 1개를 입력 받는다.
            - [x] 당첨 번호화 보너스 번호를 검증한다 (alert로 에러 메세지 띄우기)
          - [x] 결과 확인하기 버튼을 누르면 당첨 통계를 계산하고 수익률을 계산한다.
          - [x] 당첨 통계 모달 UI를 보여준다.
      3. [x] 당첨 통계 모달
          - [x] `X` 버튼, 'ESC'를 누르면 모달을 닫는다.
          - [x] 당첨 통계 타이틀 출력
          - [x] 일치 갯수, 당첨금, 당첨 갯수를 알려주는 표 출력
          - [x] 수익률을 알려주는 텍스트 출력
          - [x] 다시 시작하기 버튼 클릭
            - [x] 구입 전 화면으로 돌아간다.
            - [x] 게임 데이터 초기화, 화면 UI 초기화, 모달 닫기
